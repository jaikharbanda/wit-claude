You built it on Saturday: now use it

The three questions everyone asked at the end of the session, answered. Want the full walkthrough for every file? Open the step-by-step guide →

1 · I have an about-me.md. What do I do with it?

Nothing, and that's the point. With Prompt 1 pasted into your global instructions (Settings → Cowork), Claude reads everything in ABOUT ME/ before every task, automatically. You never attach the file or mention it. Start any task in Cowork and Claude arrives already briefed on who you are and how you work. (Free plan: the same file sits in your Project's instructions and works the same way.) Didn't finish the interview in the room? The Week 3 skill below runs it end to end.

2 · Prove it works: the A/B test

Run the same task twice and compare:

  • A (before): open plain browser Claude at claude.ai, no Project, and run Prompt 5 below. That's Claude off the shelf.
  • B (after): spin off a new task in Cowork and run the identical prompt. That's your Claude.

Read the two side by side: A is generic filler about nothing in particular; B knows your work, your clients and your tone. The order matters: the "before" must run outside Cowork, because inside it Claude reads your files and there is no before.

3 · my-company.md: set it up, then forget it's there

Set up: run Part 2 of the Week 3 skill (say "run the about-me interview, Part 2"). It's 6–8 questions, goals and decisions only: this year's targets with numbers, where your energy goes this quarter, what you're saying no to. Claude writes my-company.md (under 1,000 tokens) into ABOUT ME/.

Then what? Same as about-me: nothing. Claude reads it before every task, so every plan, draft and recommendation gets steered by your actual goals, and Claude stops suggesting things you're saying no to. Revisit it quarterly, not weekly: it's your north star, not your Tuesday deadline.

Prompt 1 Global instructions

Paste into: Settings → Cowork → Edit Global Instructions

I usually start my Cowork session by pointing you to my Cowork folder.

Before any and every single task, you must read every file in ABOUT ME/:

- about-me: it's me, who I am, what I love and hate
- anti-ai-writing-style: I hate how Claude writes, unless you write and then audit it against my anti-ai-writing-style file.
- my-company: where I work, my role.

Never read the folders OUTPUTS/ or TEMPLATES/ unless I specifically point you to a file.

Save all deliverables in OUTPUTS/ under a subfolder named after the project.

If the brief is unclear, use AskUserQuestion. Don't fill gaps with filler. Don't over-explain. Deliver the work.

Quick comparisons

The two things people always ask in week 1.

Claude vs ChatGPT

They've largely converged: on most everyday tasks the top models are close. The difference now is fit and workflow, not raw power.

Claude leans into
  • Coding and long, complex documents
  • Natural writing and tone
  • Working with your files (Cowork + Skills)
ChatGPT leans into
  • Image generation
  • Voice conversations
  • A bigger plugin and app ecosystem

Plenty of people use both. This group focuses on getting great at Claude.

Browser vs Desktop vs Claude Code
Browser Claude
Quick chat and one-off tasks, from any device.
Use for: fast answers, drafting, ideas.
Desktop (Cowork)
Works with your actual files, alongside you.
Use for: real deliverables and documents.
Claude Code
Build tools and websites, even with no coding background.
Use for: building something (week 4).
