The three files: a step-by-step guide

What to do with each file in your ABOUT ME folder: create it, use it, keep it sharp.

Claude AI Working Group · Week 3 companion

Before the files: two one-off steps

Do these once. Everything below assumes they're done.

Step 0 · The folder

Create a folder called Claude Cowork anywhere on your computer, with three subfolders inside: ABOUT ME, OUTPUTS, TEMPLATES. In the Claude desktop app, open the Cowork tab and point it at the Claude Cowork folder. Free plan: create a Project in browser Claude called "My Claude workspace" instead; your files will live in the Project's instructions.

Step 0.5 · Global instructions (the glue)

Go to Settings → Cowork → Edit Global Instructions and paste Prompt 1 from the hub. This is what makes everything automatic: it tells Claude to read every file in ABOUT ME before every task, and to leave OUTPUTS and TEMPLATES alone unless you point at them. Without this step the files just sit there.

File 1: about-me.md

Who you are, how you work, what good looks like, what you hate. Claude's briefing on you.

Create it (10–20 minutes, once)
  • Download the About-me interview skill from the hub (.skill for Cowork, .md for browser).
  • In Cowork: save the skill, spin off a new task and say "run the about-me interview". Browser: paste the .md into a chat.
  • Answer the ~20 questions like you'd talk to a colleague. Vague in = vague out. Dictate if you can; you speak faster than you type.
  • Claude compiles a condensed about-me.md (under 2,000 tokens) and saves it in ABOUT ME/. Browser: copy the output into your Project's instructions.
Use it

Nothing to do: with global instructions in place, Claude reads it before every task, automatically. You never attach it or mention it. Every draft, plan and answer starts from who you actually are instead of a generic user.

Test it

Run the A/B test (Prompt 5 on the hub): same prompt once in plain browser Claude with no Project, once in a fresh Cowork task. The "before" must run outside Cowork; inside it, Claude reads your files and there is no before.

Maintain it

If it grows past ~2,000 tokens (roughly 1,500 words), run Prompt 2 ("trim a bloated about-me file"). Rerun the interview every few months, or just tell Claude "update my about-me.md, here's what changed."

File 2: anti-ai-writing-style.md

The rules that stop Claude writing like a robot: banned words, banned patterns, no em dashes.

Create it (5 minutes)
  • Download the Anti-AI writing style skill (Week 1's giveaway) from the hub.
  • Put the .md file in ABOUT ME/. Browser: paste it into your Project's instructions below your about-me.
  • Make it yours: add 2–3 words that make you cringe to the banned list, delete any rule you don't care about. A hand-me-down file is good; your file is better.
Use it

Automatic, same as about-me. Claude writes, then audits its own draft against your rules before you see it. To clean up something written elsewhere, paste it in and say "audit this against my anti-ai-writing-style file."

Maintain it

It's a living file. Every few months, reread your last handful of AI drafts; add new words or patterns that now feel robotic, delete ones that no longer trip you up.

File 3: my-company.md

Goals, strategy and decisions only. Your north star, not your Tuesday deadline.

Create it (5–10 minutes)
  • Spin off a new task and say "run the about-me interview, Part 2" (same Week 3 skill).
  • Answer 6–8 questions: this year's targets with numbers, where your energy goes this quarter, what you're actively saying no to.
  • Claude writes my-company.md (under 1,000 tokens) into ABOUT ME/.
Use it

Automatic. The difference shows up in judgement calls: plans get prioritised against your actual goals, drafts point at what you're building this quarter, and Claude stops suggesting things you're saying no to. Ask it "what should I focus on this week?" and compare the answer with and without the file if you want proof.

Maintain it

Revisit quarterly, or when strategy genuinely shifts. If you're editing it weekly, it has too much detail in it: day-to-day belongs in the task, not the file.

The other two folders in 30 seconds

OUTPUTS/ needs no setup: your global instructions already tell Claude to save deliverables there, one subfolder per project. Claude never reads it uninvited, so it can grow forever without costing you tokens.

TEMPLATES/ fills itself as you work: when Claude makes something you like, say "save this as a template in TEMPLATES/" (Prompt 3). Next time: "use the template in TEMPLATES/report."
